Video Frame Interpolation · Evaluation

把评测这件事，做到公平。

VFIEval 是一个本地的视频插帧模型评测平台。放入模型与视频，就能在完全一致的推理与后处理流水线下，获得感知质量指标、可视化对比与人类偏好排名。

CUDA / NPU / CPU LPIPS · VMAF · CGVQM 盲测 A/B 评测 多卡并行推理
Overview

项目概览

VFIEval 专注评测环节：推理、后处理、指标计算与可视化对比。它不训练模型，也不实现 PSNR——目标是让不同模型在同一条流水线下被公平衡量。

本地推理

在 CUDA / NPU / CPU 上运行插帧模型，支持多卡并行、混合精度与 OOM 自动降批恢复。

统一后处理

固定的 warp → mask → blend → pred 流水线，模型只需输出光流与掩码，保证跨模型公平对比。

感知质量指标

LPIPS (ViT / ConvNeXt)、VMAF、CGVQM 四种客观指标，覆盖帧级感知距离与视频级主观质量。

双轨对比 Compare

同一 GT 与 1–2 份 Pred 严格对比，逐帧并排查看 flow / mask / warp 可视化层。

盲测 Campaign

匿名 A/B 评测，多人评审 + Bradley-Terry 排名与 bootstrap 置信区间。

评分与反馈统计

为每个 Run 记录 1–5 分评分与文字问题，按视频 / 模型 / checkpoint 聚合分析。

设计原则：模型的职责被压缩到最小——只输出光流与掩码 logits；warp、blend、compose 全部由平台统一执行。所有模型因此站在同一条起跑线上，指标差异只反映模型本身。

Workflow

评测流程

从放入文件到查看评测结果的完整路径。标准推理是主线，对比与盲测是可选的进阶评测模式（见下文「对比与盲测」）。

2

Contract

模型接口

模型只需实现一个 infer(img0, img1)，输出光流和掩码 logits，平台负责其余所有后处理。

class Model: def __init__(self, checkpoint_path=None, device="cpu", metadata=None): # 加载权重（可选） ... def infer(self, img0, img1): # img0, img1: [B, 3, H, W] RGB float [0,1], 已在 device 上 return { "flowt_0": flowt_0, # [B, 2, h, w] backward flow "flowt_1": flowt_1, # [B, 2, h, w] backward flow "mask0": mask0, # [B, 1, h, w] logits "mask1": mask1, # [B, 1, h, w] logits }
class Model: def infer(self, img0, img1): # 按顺序返回 4 个张量 return flowt_0, flowt_1, mask0, mask1

输入 / 输出规格

参数形状说明
img0 / img1[B, 3, H, W]RGB 浮点张量，值域 [0,1]，已 resize 到推理分辨率，在目标 device 上
flowt_0 / flowt_1[B, 2, h, w]Backward flow（像素坐标系），分辨率可低于输入
mask0 / mask1[B, 1, h, w]Logits（不能提前 sigmoid），分辨率可低于输入
契约约束
  • mask 必须是 logits，不要 sigmoid——平台统一处理。
  • flow 使用 target_pixel + flow 约定（backward flow）。
  • 输出低分辨率时，flow 会按 W/w 和 H/h 缩放位移后 resize 到输入分辨率。
  • 输出必须与输入同 batch、同 device、同 dtype。
Post-processing

后处理流水线

平台用一条固定的流水线，把模型输出的光流和掩码合成为最终插帧。此流程对所有模型一致、不可覆盖——这是「公平对比」的技术前提。

点击任意节点，可在下方公式列表中高亮对应计算步骤。

计算步骤

1Backward Warp
warp0 = grid_sample(img0, grid + flowt_0, mode='bilinear', padding='border', align_corners=True)
2Backward Warp
warp1 = grid_sample(img1, grid + flowt_1, mode='bilinear', padding='border', align_corners=True)
3Mask Sigmoid
mask0 = σ(mask0_logits) mask1 = σ(mask1_logits)
4Blend
blend = mask0 · warp0 + (1 − mask0) · warp1
5Compose
pred = mask1 · img1 + (1 − mask1) · blend
6Clamp
output = clamp(pred, 0, 1)
grid_sample 参数说明
参数值说明
modebilinear双线性插值采样
padding_modeborder边界像素填充
align_cornersTrue角点对齐

低分辨率 flow 输出时，平台先按 W/w 和 H/h 缩放位移值，再 resize 到输入分辨率后执行 warp。

Metrics

评测指标

四种客观质量指标，覆盖帧级感知距离与视频级主观质量。先理解「为什么不用 PSNR」，再逐个拆解每个指标怎么算、适合什么视频。

为什么不用 PSNR / SSIM？

插帧模型经常把物体「放到差不多对的位置」。PSNR 逐像素求差，物体只要平移一两个像素，误差就急剧放大——即使人眼看上去几乎完美。SSIM 关注局部结构，但对插帧最容易出问题的 形变、鬼影、时间抖动 并不敏感。VFIEval 因此不实现 PSNR，转而采用两类更贴近人眼的指标：

  • 感知特征距离（LPIPS 家族）：在深度网络的特征空间里比较，对轻微位移宽容，对纹理 / 结构失真敏感。
  • 学习型视频质量（VMAF / CGVQM）：直接拟合人类主观打分，并把「运动」和「时间维度」纳入考量。

lpips_vit_patch — DINOv2 Patch Feature Distance

VMAF — Video Multi-Method Assessment Fusion

Netflix 开源的全参考视频质量指标。核心思想：不用一条数学公式去猜质量，而是先抽取若干互补的画质特征，再用一个在真实人眼评分上训练过的模型，把这些特征「融合」成一个 0–100 的分数——尽量逼近真人打分。

每帧算出这几个特征，喂进一个用主观评分（MOS）训练的 SVR 回归模型，得到该帧 VMAF；再对所有帧取均值（pooling）得整段视频分数。时间特征的加入是 VMAF 区别于纯图像指标的关键——它知道画面在动，也知道人眼在运动场景下对某些瑕疵没那么苛刻。

对什么样的视频最有价值？
  • 高运动 / 影视实拍内容：VMAF 训练数据以流媒体实拍为主，判断运动模糊、压缩、缩放伪影很准。
  • 关心「观感」而非「像素精度」：插帧要的是流畅自然，VMAF 直接对齐主观体验。
  • 整段视频质量对比：给出视频级分数，适合横向比较不同模型的整体表现。
使用前提与局限
  • 需要 GT 与 Pred 逐帧对齐、分辨率一致的两段视频，不支持单帧图片。
  • 训练数据偏向自然实拍——对动画、屏幕内容、合成图形等分布外内容相关性下降，建议配合 LPIPS / CGVQM 一起看。
  • 本平台使用默认 VMAF 模型；4K 内容如需更准的分数，可自行切换官方 4K 模型。

CGVQM — Computer Graphics Video Quality Metric

Intel Labs 提出的学习型全参考视频质量指标，用 3D 卷积网络在时空维度上比较 GT 与 Pred，专门针对渲染 / 合成类失真（鬼影、闪烁、时间抖动、上采样伪影）训练——这些恰恰是插帧最常见的问题。分数越低表示与参考越接近，是对 VMAF（偏自然实拍）的有益补充。

如何选择指标？

指标粒度方向最擅长建议场景
LPIPS-ViT逐帧↓ 越低越好整体结构 / 语义一致性想定位「哪一帧崩了」，看逐帧曲线与最差帧

CGVQM视频级↓ 越低越好时间抖动 / 鬼影 / 合成类失真动画、渲染、上采样内容，或 VMAF 相关性不足时补充

Evaluation Modes

对比与盲测

除标准推理外，平台提供两种进阶评测模式：客观的双轨可视化对比，以及引入人类偏好的匿名盲测。

双轨对比 Compare

从同一个 Media Item 选择 GT 与 1–2 份可复用 Pred，逐帧并排对比，并按需叠加 flow / mask / warp 可视化层。

Item-first，最多双 Pred

一个 GT Item 对比一至两个属于该 Item 的 Pred 成员，拒绝跨 Item 组合。

可视化层

并排查看 flow / mask / warp / blend，直观定位模型在哪一步出了问题。

严格时间身份

源帧顺序、帧数、FPS 与可用时间戳必须一致；空间尺寸按记录指纹的 LANCZOS Alignment Plan 显式归一化。

盲测 Campaign

匿名 A/B 评测，把「哪个模型更好」交给人来判断，得到客观可信的人类偏好排名。

1

创建 Campaign

在 Evaluation Studio 中选择两个已完成 Run 的 Track 作为方法 A 和 B。

2

确认视频矩阵

系统展示两方法共有视频的对齐情况（帧数、尺寸、FPS 必须严格一致）。

3

发布评测

冻结 GT / A / B 视频到评测包并生成参与链接，之后删除源 Run 也不影响评测。

4

参与者评审

匿名界面，wipe 对比或完整视图，投票 A 更好 / B 更好 / 平局，可选填评分（1–5，0.25 步长）。

5

统计分析

Bradley-Terry 排名 + 固定种子 bootstrap 置信区间，平局计半胜。客观与主观指标分开展示。

隐私保护：参与者只看到匿名的「方案 A」和「方案 B」，不暴露模型名称、checkpoint、Run ID 或真实方法标签。左右随机分配且稳定。
Performance

性能与架构

三级流水线并行设计，让设备在等待 CPU 工作时不空转，最大化利用率。

多卡分片

优先按视频拆分到不同设备；视频不足时将长视频切为连续 sample segment 分片。各 shard 独立写帧，finalize job 统一合并视频。

指标缓存

SHA-256 key 绑定 metric 名称、适配器版本、文件身份（path + size + mtime + sha256）与设备。更换权重或版本后自动失效旧缓存。

OOM 恢复

GPU 内存不足时自动将 batch size 减半重试，最低降到 1。不回退 CPU——失败时明确报告设备与原因。

进度与取消

实时进度更新（每 max(1, total/200) 样本写入一次）。支持随时取消正在运行的推理或评测任务。
